# Sentinel — Self-Healing Agentic RAG

Guardrailed, self-healing RAG platform: FastAPI + LangGraph on AWS Lambda (free tier),
hybrid FAISS+BM25 retrieval, Groq LLMs, critic/repair loop, full request tracing.

## Status
- [x] P1 — ingestion + hybrid index (`make ingest`)
- [ ] P2 — LangGraph agent (router → retriever → synthesizer → critic → repair)
- [ ] P3 — guardrails + API + tests
- [ ] P4 — AWS deploy (Lambda container, API GW, DynamoDB, S3, CloudFront)
- [ ] P5 — evals + CI
- [ ] P6 — frontend

## Quickstart (local)
```bash
python -m venv .venv && make install   # Python 3.12 required
cp .env.example .env                   # fill GROQ_API_KEY
make ingest                            # build index/ from ./docs, runs smoke test
make test
```

## Architecture
(diagram + trade-offs table land in P2/P4)

| Decision | Choice | Why |
|---|---|---|
| Vector store | FAISS IndexFlatIP in Lambda memory, artifacts in S3 | exact search, zero servers, free tier |
| Sparse + fusion | BM25 + RRF (k=60) | exact-term recall; rank fusion needs no score calibration |
| LLM | Groq (8b routing/critic, 70b escalation) | free tier, fast inference |
| Auth | API GW usage plans + hashed keys in DynamoDB | two layers, no Cognito |
